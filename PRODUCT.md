# Product

## Register

product

## Users

นักศึกษา, อาจารย์, และ admin ในสภาพแวดล้อมมหาวิทยาลัย

**Context:** ใช้งานบน desktop/laptop ในวิทยาเขต หรือ remote เพื่อจองห้องผลิตสื่อดิจิทัล

**Job to be done:**
- นักศึกษา/อาจารย์: จองห้องผลิตสื่อดิจิทัล, ตรวจสอบสถานะการจอง, ยกเลิกการจอง
- Admin: อนุมัติ/ปฏิเสธการจอง, จัดการผู้ใช้, จัดการห้องและวันหยุด

## Product Purpose

ระบบจองห้องผลิตสื่อดิจิทัลสำหรับมหาวิทยาลัย เพื่อให้นักศึกษาและอาจารย์สามารถจองห้องได้อย่างมีประสิทธิภาพ ลดการขัดแย้งเรื่องเวลา และให้ admin จัดการการจองได้ง่าย

**Success:** การจองที่รวดเร็ว ชัดเจน และลดการซ้ำซ้อนของการใช้ห้อง

## Brand Personality

เป็นทางการ, เรียบง่าย

**Voice & Tone:** สุภาพ ชัดเจน ไม่ซับซ้อน เน้นประสิทธิภาพ

**Emotional Goals:** มั่นใจในระบบ สะดวกในการใช้งาน ไม่สับสน

## Anti-references

- ดูเก่า (outdated design)
- ซับซ้อนเกินไป (overly complex)
- ดูเหมือน SaaS ทั่วไป (generic SaaS look)

## Design Principles

1. **ความชัดเจนก่อนความสวยงาม** - ข้อมูลสำคัญต้องอ่านง่าย เข้าใจได้ทันที
2. **เรียบง่ายแต่มีประสิทธิภาพ** - ลดขั้นตอนที่ไม่จำเป็น แต่ยังคงฟังก์ชันครบถ้วน
3. **เป็นทางการแต่ไม่เคร่งเครียด** - ดูมืออาชีพ เหมาะกับสภาพแวดล้อมมหาวิทยาลัย

## Accessibility & Inclusion

**Standard:** WCAG 2.1 Level AA

**ข้อกำหนดเฉพาะ:**
1. **Color Contrast:** อัตราส่วนความต่างของสีระหว่างตัวอักษรและพื้นหลังต้องไม่ต่ำกว่า 4.5:1 เพื่อให้ผู้ใช้สายตาเลือนรางหรือผู้สูงอายุอ่านง่าย
2. **Keyboard Navigation:** สามารถใช้ปุ่ม Tab และ Arrow Keys ในการเลื่อนดูและกดปุ่มต่างๆ บนหน้าเว็บได้ 100% โดยต้องมี Focus State (เส้นขอบ) ที่มองเห็นชัดเจนเมื่อกดเลือกสิ่งนั้น
3. **Screen Reader Friendly:** โครงสร้างโค้ดต้องรองรับโปรแกรมอ่านหน้าจอสำหรับผู้พิการทางสายตา (มี alt text สำหรับรูปภาพ และใช้ Semantic HTML อย่างถูกต้อง)
4. **ไม่ใช้ "สี" เป็นสิ่งเดียวในการสื่อสารความหมาย** (เช่น ฟอร์มที่กรอกผิด ต้องมีทั้งสีแดงและข้อความเตือนชัดเจน)
