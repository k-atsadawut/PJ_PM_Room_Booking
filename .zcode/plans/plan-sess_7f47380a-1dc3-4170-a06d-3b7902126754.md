## ภาพรวม

เปลี่ยน flow "ลืมรหัสผ่าน" จากการรอ admin อนุมัติ → เป็นยืนยันตัวตนผ่าน OTP ด้วยตัวเอง ทุกขั้นตอนเป็น popup กลางจอบนหน้า `forgot-password.html` หน้าเดียว ไม่ต้องเปลี่ยนหน้า

## Flow ใหม่ (forgot-password)

```
หน้า forgot-password.html
  └─ popup กลางจอ (multi-step)
       Step 1: กรอกอีเมล → กดส่ง → backend ส่ง OTP ไปอีเมล + แจ้งเตือน admin
       Step 2: popup เปลี่ยนเป็นช่องใส่ OTP (6 หลัก) + ปุ่มส่งใหม่
       Step 3: OTP ถูก → popup เปลี่ยนเป็นฟอร์มเปลี่ยนรหัสผ่าน
       Step 4: เปลี่ยนสำเร็จ → popup แสดงข้อความสำเร็จ → ปุ่มกลับ login
```

## ส่วนที่ 1: Database — เพิ่ม OTP type

**ไฟล์**: `database/schema.sql` (บรรทัด 150)

`otp_codes.Type` enum ตอนนี้มี `'password_change','email_verification'` → เพิ่ม `'password_reset'`:
```sql
Type ENUM('password_change','email_verification','password_reset')
```

**Migration SQL** (รันบน production DB):
```sql
ALTER TABLE otp_codes
MODIFY COLUMN Type ENUM('password_change','email_verification','password_reset')
NOT NULL DEFAULT 'password_change';
```

UI แต่ละ step:
- **email**: input email + ปุ่มส่ง
- **otp**: 6 ช่อง + ปุ่มยืนยัน + ปุ่มส่งใหม่ (countdown 60 วินาที)
- **reset**: input รหัสใหม่ + ยืนยันรหัส + ปุ่มบันทึก
- **done**: ข้อความสำเร็จ + ปุ่ม "กลับหน้า login"

ลาก pattern จาก `change-password.html` และ popup ใน `dashboard.html:270`

## ส่วนที่ 4: Admin tab (ลดขั้นตอน)

**ไฟล์**: `frontend/admin/dashboard.html` — `PasswordRequestsTab`

ตอนนี้ admin ต้องกด approve/reject → เปลี่ยนเป็น **แค่แสดงประวัติ** (read-only) ไม่มีปุ่ม approve/reject แล้ว เพราะ flow ใหม่ไม่ต้องอนุมัติ

แต่ยังเก็บ record ใน `password_reset_requests` เป็น audit log (status = 'auto-processed')

> ถ้าอยากลบ tab นี้ทิ้งเลยก็ได้ บอกได้

## ขอบเขต (ไม่ทำในเฟสนี้)

- **ไม่แตะ `change-password.html`** (force change ตอนเข้าครั้งแรก) — คงเป็นฟอร์มเดิม
- **ไม่เพิ่ม rate limiting** สำหรับ forgot-password (security improvement แยกทีหลัง — ตอนนี้ใช้ OTP expiry 10 นาที + cap สั้น ๆ พอ)
- **ไม่ลบ `password_reset_requests` table** — เก็บเป็นประวัติ

## ไฟล์ที่จะแก้ (4 ไฟล์)

1. `database/schema.sql` — เพิ่ม enum value `'password_reset'` (+ migration SQL ให้รัน)
2. `src/routes/password-reset.js` — แก้ POST `/` + เพิ่ม `/verify-otp` + `/reset`
3. `frontend/forgot-password.html` — เขียนใหม่เป็น multi-step popup
4. `frontend/admin/dashboard.html` — `PasswordRequestsTab` เปลี่ยนเป็น read-only (หรือลบ tab ทิ้ง ถ้าคุณเลือก)

## วิธีตรวจรับ

1. รัน migration SQL บน DB ก่อน
2. Deploy → ทดสอบ flow เต็ม: login → ลืมรหัส → กรอกอีเมล → ได้รับ OTP → ใส่ OTP → เปลี่ยนรหัส → login ด้วยรหัสใหม่ได้
3. ทดสอบ security: ใส่ OTP ผิด → ต้องติด error, token หมดอายุ 5 นาที
4. ตรวจว่า admin ได้รับ notification (ในระบบ)
